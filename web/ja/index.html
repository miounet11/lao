AI エージェント、MCP、ターミナル向けのブラウザ自動化

実際に使っている Chrome をローカル Browser API にする。

iatlas-browser は新しいヘッドレスブラウザを起動するのではなく、 すでに使っている Chrome セッションをそのまま扱います。ログイン状態、Cookie、タブ、 ページ状態を維持したまま自動化できます。

install.sh を取得 拡張機能 zip Open API docs GitHub

クイックインストール

curl -fsSL https://miaoda.vip/install.sh | bash
  • ローカルにインストール
  • CLI と拡張機能をビルド
  • MCP と API の設定ファイルを生成

Open API

メールで API キーを発行

https://miaoda.vip/v1/ でホスト API を利用できます。メールにひもづく API キーごとに、 1 日あたり 1000 リクエスト まで使えます。

API 概要

  • 認証: `Authorization: Bearer <apiKey>`
  • 上限: 1 日 1000 リクエスト
  • エンドポイント: `POST /v1/register`, `GET /v1/usage`, `POST /v1/open`

何が便利か

ログイン済みサイトの自動化に強い

既存の認証状態を再利用

社内ツール、管理画面、SaaS ダッシュボードなど、再ログインが面倒なサイトに向いています。

AI ツールへの接続が簡単

MCP モードとローカル API があるため、エージェントから実ブラウザを扱う入口を作りやすいです。

最初の手順

セットアップの流れ

1. インストール

curl -fsSL https://miaoda.vip/install.sh | bash

2. 拡張機能を読み込む

chrome://extensions/

~/.iatlas-browser/extension を指定します。

3. デーモン起動

iatlas-browser daemon
iatlas-browser doctor

4. コマンド実行

iatlas-browser open https://example.com
iatlas-browser snapshot -i

接続方法

CLI、MCP、ローカル API

CLI

ブラウザ操作、スナップショット、クリック、ネットワーク確認などを直接実行できます。

MCP

Cursor や Claude Desktop 用に使える MCP 設定を出力できます。

cursor.json

API

ローカル HTTP API を使って外部スクリプトから制御できます。

examples.sh
