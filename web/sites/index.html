adapter catalog + hosted extensions

Browse every callable site interface from miounet11/lao-s and the hosted extension layer.

This page exposes the full adapter catalog used by iatlas-browser site. You can search by platform, inspect arguments, see whether a command is hosted or local-only, copy the exact CLI command, copy an MCP request payload, and call the hosted subset directly through the API. Additional hosted-only extensions are included for high-value public sources that are safe to run on miaoda.vip.

View lao-s Catalog API Open API docs

How to use these adapters

  • CLI: iatlas-browser site <name> ...
  • MCP: use site_list, site_search, and site_run
  • Hosted API: use /v1/sites/hosted and /v1/sites/run
  • Catalog API: query /v1/catalog/sites and /v1/catalog/site

Hosted adapters run directly on miaoda.vip. Local-only adapters still require your own iatlas-browser runtime and browser session. That local path remains the primary adapter model.

Hosted execution

These adapters run directly on the remote server. They are public, read-only, and do not depend on a personal logged-in browser session. 

curl -s https://miaoda.vip/v1/sites/run \
  -H "Authorization: Bearer $API_KEY" \
  -H "Content-Type: application/json" \
  -d '{"name":"github/repo","args":{"repo":"miounet11/lao"}}'

Local runtime / MCP

Adapters that depend on your own login, browser state, tabs, or site-side stores remain local by design. Use the same adapter name through CLI or the MCP site_run tool. 

{
  "tool": "site_run",
  "arguments": {
    "name": "twitter/search",
    "args": {
      "query": "claude code"
    }
  }
}
